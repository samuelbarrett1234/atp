4 /*
5 
6 EquationalKnowledgeKernel.h
7 
8 Implementation of the IKnowledgeKernel for equational logic. This
9 object contains the functions which tell you what is/isn't allowed
10 in equational logic.
11 
12 More specifically, the successor statements in equational logic are:
13 - Replacing either side of an equation with one of the given
14  equality rules (just transitivity in disguise; if x=y is
15  our statement, and we have a rule of the form y=z, then we
16  can turn the statement into x=z.)
17 - Replacing a free variable with any of: a user-defined
18  constant, a user-defined function whose arguments are new
19  free variables, or a free variable which already exists
20  within the statement.
21 For example, with the latter rule, if our statement is
22 "f(x)=g(x)" then we can obtain:
23 "f(e)=g(e)" for user defined "e", and "f(h(x))=g(h(x))" for
24 user defined "h", and instead if our statement is
25 "f(x, y)=g(h(x), y)" then "f(x, x)=g(h(x), x)".
26 
27 */

102  // precondition: is_defined(name)

104  // should be assumed that if two symbol names agree on their
105  // hash, they agree on their symbol name too.
106  inline size_t symbol_id(std::string name) const

108  ATP_LOGIC_PRECOND(is_defined(name));
109  const size_t id = std::hash<std::string>()(name);
