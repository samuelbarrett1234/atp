Detailed Description

IKnowledgeKernel implementation for equational logic.

Basically acts as an optimised expression-matcher. Note: every time you call add_theorems or remove_theorems, it forces a "matching rule rebuild", which is an O(n^2) operation in the total number of theorems and axioms, and it builds a set of expression matching rules to make it efficient to iterate over successor statements etc.

Characterises the information provided to this knowledge kernel in a single integer code.

If two knowledge kernels agree on their code, then they agree on the return results of all the functions below. (This code is computed similarly to a hash code of all the user definitions, axioms, and available theorems.)

bool atp::logic::equational::KnowledgeKernel::try_match ( size_t  match_index,

FreeVarMap< Expression > *  p_out_subs 
) const

Try to match the given expression to the match rules at the given index.

The match does not attempt to apply rules recursively; instead it tries to apply the rules at the root of expr.

Parameters
match_indexThe matching rule to try
exprThe expression to try matching
p_out_subsAn optional output parameter, for extracting the matching if successful.
Returns
True if and only if the match was successful.
Postcondition
p_out_subs, if not null, will be given a mapping which is total with respect to the matching expression's free variables - but not necessarily the match results.
Precondition
match_index < num_matching_rules()
