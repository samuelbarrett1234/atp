Detailed Description

Implementation of the IStatement interface for equational logic.

Warning
This class implements an arity limit on the functions, for efficiency.
Note
As part of the optimisation, we store a structure almost like a table of function nodes, rather than a syntax tree.
As much as this class is supposed to be immutable, in order to be usable in standard containers like std::vector, it needs a copy constructor and an assignment operator.
Statement objects are tied to a model context, but not to any particular knowledge kernel.

Perform a fold over pairs of statements, where the fold constructor is dependent on the node types of the pair. There are functions for pairs of matching types, and then a default function for when the pairs do not match.

Template Parameters
ResultTThe return type (all functions must return this type.)
EqPairFuncTThe function operating on pairs of equality nodes, must have signature ResultT x ResultT -> ResultT
FreePairFuncTThe function operating on pairs of free variable nodes, must have signature size_t x size_t -> ResultT where the two given integers are the free variable IDs in the pair.
ConstPairFuncTThe function operating on pairs of constants, must have signature size_t x size_t -> ResultT where the two integers are the symbol IDs of the constants
FuncPairFuncTThe function operating on pairs of function nodes, must have signature size_t x size_t x std::vector< ResultT>::iterator x std::vector<ResultT>::iterator -> ResultT where the two integers are the symbol IDs of the functions and the two iterators are the begin and end iterators of the function arguments' results.
DefaultPairFuncTThis is called when a pair of nodes are encountered but don't have the same type, and it must have signature SyntaxNodePtr x SyntaxNodePtr -> ResultT, where the two arguments are just the two nodes.
Parameters
otherThe other statement to fold with, which can be thought of as appearing on the RHS of the pair fold.

Statement atp::logic::equational::Statement::increment_free_var_ids ( size_t  inc) const

Add a particular value to the ID of every free variable in this statement.

This is useful for ensuring the free variable IDs of two statements do not clash, for example if we know the largest ID in one statement, we can increment the IDs in this statement by that amount + 1, then they won't clash.
