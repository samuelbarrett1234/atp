Parameters
num_loadThe number of theorems to load from the database
num_returnThe number of theorems to return from done, which will be a subset of the theorems loaded from the database (note that we could return less than this, as there are no guarantees of this many being present in the database in the first place.)
symbol_mismatch_costThe edit distance cost for a mismatch of symbols, relative to the cost of a free variable substitution which is always fixed at 1.0
symbol_match_benefitThe corresponding benefit (decrease in edit distance) incurred when two symbols match
weightingHigher means a statement is considered "better" if it is useful in lots of places; lower means a statement is considered "better" if it is very useful in a handful of places. Must be > 0.
Precondition
symbol_mismatch_cost > 0 and symbol_match_benefit > 0 and weighting > 0
num_load >= num_return
