Detailed Description

This is a naive, simple stopping strategy: always keep N successors in memory, and always just return the best one.

This strategy is naive because it pays no attention to cost and is hardly even a stopping strategy. However, its simplicity makes a great benchmark (if a complex strategy cannot beat this then it should be discarded).
