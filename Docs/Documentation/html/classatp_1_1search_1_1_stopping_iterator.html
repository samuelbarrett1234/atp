Detailed Description

Wraps around an iterator by extracting several successors at a time from it, and returning the "best" one seen so far.

See also
atp::logic::IPfStateSuccIter

Computation of successors can be expensive, but the order in which they are evaluated is not necessarily optimal. This iterator tries to return successors in (approximately) non- increasing order of "benefit". It does so by extracting several and returning the best. However, extraction is expensive, so there is a tradeoff to be struck between extracting lots of successors vs returning the best seen so far. This is an optimal- stopping problem.
